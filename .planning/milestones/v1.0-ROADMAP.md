# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-03
**Phases:** 1–2
**Total Plans:** 5

## Overview

A modular weather application built with Streamlit that allows users to search any city and view real-time weather data including temperature, conditions, humidity, wind speed, and wind direction. Features an interactive dark-themed map with city markers.

## Phases

### Phase 1: Core Weather Features

**Goal:** Users can search for any city and view current weather data (temperature, conditions, humidity, wind)
**Depends on:** Nothing (first phase)
**Plans:** 2 plans

Plans:
- [x] 01-01: Foundation (models, utils, services)
- [x] 01-02: UI Layer (Streamlit app)

**Details:**

Requirements: CITY-01, CITY-02, TEMP-01, TEMP-02, TEMP-03, COND-01, COND-02, METR-01, METR-02, METR-03, ERR-01, ERR-02

Success Criteria (all met):
1. User can search for a city by name and see weather results
2. User sees "city not found" error when searching an invalid city
3. User can view current temperature in Celsius
4. User can toggle between Celsius and Fahrenheit; preference persists during session
5. User can view current weather condition as human-readable text
6. User can view humidity percentage
7. User can view wind speed
8. User can view wind direction as cardinal direction (N, NE, E, etc.)
9. App displays user-friendly error messages when API calls fail
10. App displays user-friendly error messages when network errors occur

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Streamlit framework | Fastest path to working web UI | ✓ Good |
| Open-Meteo API (free) | No API key, good global coverage | ✓ Good |
| UV package manager | Modern, fast Python package manager | ✓ Good |
| Dataclasses with `__post_init__` validation | Type safety without full Pydantic overhead | ✓ Good |
| `@cache_data` with 300s TTL | Prevent redundant API calls | ✓ Good |
| CartoDB dark_matter tiles | Matches dark theme aesthetic | ✓ Good |
| CSS injection for gradient bg + centering | Streamlit config.toml doesn't support gradients | ✓ Good |
| Folium `folium.Element` for attribution CSS | Attribution lives in iframe, unreachable from Streamlit CSS | ✓ Good |

**Issues Resolved:**

- WeatherData model was missing latitude/longitude (auto-fixed in 02-03)
- Container width was too narrow (400px → removed max-width, let Streamlit default)
- C/F buttons stacked vertically when container was narrow (fixed by width fix)
- Leaflet attribution bar hidden via CSS injection into iframe

**Technical Debt Incurred:**

- None significant
